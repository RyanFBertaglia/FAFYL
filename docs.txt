# Contexto do Projeto: FAFYL — Recomendador de Cursos
## Visão geral
FAFYL é um sistema de recomendação de cursos universitários baseado em perfil DISC do usuário. O usuário responde um quiz com perguntas de múltipla escolha; cada alternativa mapeia para uma dimensão DISC (D, I, S, C) com um peso. O sistema calcula o perfil DISC do usuário e usa produto escalar (dot product) contra os pesos DISC de cada curso, retornando uma lista ordenada por afinidade. O nome do projeto vem do mascote da aplicação, o Capelinho — um personagem visual que muda de aparência de acordo com o curso ou preferência do usuário, configurado manualmente pelo frontend.
---
## Stack
- Backend: Java com Spring Boot, Spring Security (JWT), Spring Data JPA
- Banco de dados: Relacional (schema abaixo), com campos JSON para listas e coordenadas
- Autenticação: JWT sem refresh token; existe um endpoint dedicado que renova o token usando o próprio access token ainda válido

## Fluxo de autenticação
### Login — POST /auth/login
1. Cliente envia { email, password }
2. Controller busca o User por email no banco
3. Valida senha
4. Chama JwtService.generateToken(user) → JWT assinado
5. Retorna { token }
### Requisição autenticada
- Todo endpoint protegido passa pelo filtro do Spring Security
- O filtro chama JwtService.validateToken(jwt) → extrai claims
- Se válido, injeta o userId no contexto de segurança
### Renovação de token — POST /auth/renew
1. Cliente envia o access token ainda válido no header
2. Backend valida o token atual (deve estar válido — não aceita expirado)
3. Gera um novo JWT com nova expiração
4. Retorna { novoToken }
> Não existe refresh token separado. A renovação usa o próprio access token enquanto ainda não expirou.

## Endpoint principal — GET /fafyl
### Input (corpo da requisição)
```json
{
  "discProfile": {
    "D": 0.8,
    "I": 0.3,
    "S": 0.5,
    "C": 0.2
  }
}
```
> O discProfile é calculado a partir das respostas do quiz. Cada alternativa escolhida contribui com seu weight para a dimensão correspondente. O userId é extraído do JWT no header.
### Lógica do FafylService (pipeline em stream)
```java
stream<Course>
  // 1. Scoring: produto escalar entre discProfile do quiz e disc_weights do Course
  .map(course -> new Fafyl(course.dotProduct(quiz.getDiscProfile()), course))
  // 2. Descarta cursos sem nenhum match (score <= 0)
  .filter(fafyl -> fafyl.score() > 0)
  // 3. Ordena do maior para o menor score
  .sorted(Comparator.comparingDouble(Fafyl::score).reversed())
  .toList()
```
*Course.dotProduct(Map<Character, Double> profile)**
```java
public double dotProduct(Map<Character, Double> profile) {
    if (discWeights == null) return 0.0;
    double score = 0.0;
    for (Map.Entry<Character, Double> entry : discWeights.entrySet()) {
        double profileValue = profile.getOrDefault(entry.getKey(), 0.0);
        score += entry.getValue() * profileValue;
    }
    return score;
}
```
Retorna o produto escalar entre o perfil DISC do usuário e os pesos DISC do curso.
*Fafyl** é um DTO com { double score, Course course }.
### Após o pipeline
1. Salva o histórico automaticamente: { userId, course_id, accessed_at }
2. Retorna 200 OK com List<Fafyl> ranqueada
### Participantes do fluxo
```
Cliente → Controller → FafylService → CourseRepo (streamAll)
                    ↘               ↘ HistoricoRepo (save)
```

## Regras de negócio resumidas
| regra | detalhe |
|---|---|
| Ranking por score descendente | Maior produto escalar DISC = melhor posição |
| Score > 0 filtra após scoring | Cursos sem compatibilidade DISC não aparecem no resultado |
| Histórico é snapshot imutável | Salvo automaticamente; registra o curso recomendado e a data |
| Capelinho é cosmético | Não influencia nenhum cálculo; escolhido manualmente pelo usuário no frontend |
| disc_weights: Mapa JSON com pesos D, I, S, C | Cada curso tem um perfil DISC ideal representado por pesos entre 0.0 e 1.0 |
| Quiz DISC: Perguntas com alternativas mapeadas a dimensões | Cada alternativa tem dimension (D/I/S/C) e weight (0.0-1.0) |
| Perfil do usuário: Soma dos weights das alternativas escolhidas | O discProfile é o acumulado dos pesos por dimensão |
